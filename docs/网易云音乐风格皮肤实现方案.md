# 网易云音乐风格皮肤实现方案

## 一、设计风格定位

### 1.1 目标风格
参考网易云音乐桌面端的视觉风格，打造深色沉浸式界面。

### 1.2 核心特征

| 特征 | 方案 |
|------|------|
| 主题色 | 红色 `#c20c0c` / `#e60026` |
| 背景色 | 深色系 `#1a1a1a` ~ `#2d2d2d` |
| 侧边栏 | 保留文字（宽度 200px） |
| 顶部栏 | 有（搜索/面包屑/主题信息） |
| 底部栏 | 无 |
| 毛玻璃 | 无 |

### 1.3 布局结构

```
┌─────────────────────────────────────────────────────────┐
│  顶部栏 (48px) - 面包屑导航 / 搜索框 / 用户信息           │
├──────┬──────────────────────────────────────────────────┤
│      │                                                  │
│  侧  │                                                  │
│  边  │                                                  │
│  栏  │              主内容区                             │
│200px │                                                  │
│      │                                                  │
│      │                                                  │
└──────┴──────────────────────────────────────────────────┘
```

## 九、样式与业务逻辑分离保障机制

本方案通过以下设计确保样式改动不影响业务逻辑代码：

### 9.1 CSS 变量抽象层

所有颜色、间距、圆角等样式值通过 CSS 变量统一定义，业务组件只引用变量名：

```css
/* ✅ 正确做法 - 使用变量 */
.button {
  background: var(--theme-primary);
}

/* ❌ 禁止做法 - 硬编码 */
.button {
  background: #c20c0c;
}
```

### 9.2 文件职责分离

| 文件 | 职责 | 改动风险 |
|------|------|----------|
| `themes/netease-dark.css` | 仅定义 CSS 变量值 | 低，纯样式文件 |
| `*.vue` 组件 | 引用 CSS 变量，不定义颜色 | 业务逻辑不受影响 |
| `stores/theme.ts` | 主题状态管理 | 独立模块，与业务解耦 |

### 9.3 分层架构

```
┌─────────────────────────────────────┐
│  样式层 (CSS 变量定义)              │  ← 皮肤改动只影响这里
├─────────────────────────────────────┤
│  展示层 (组件模板，引用变量)         │  ← 不改动
├─────────────────────────────────────┤
│  逻辑层 (脚本代码)                   │  ← 完全不涉及样式
└─────────────────────────────────────┘
```

### 9.4 强制规范（推荐）

为确保团队成员遵循规范，建议：

1. **Code Review 检查**：禁止在组件中硬编码颜色值
2. **Stylelint 规则**（可选）：
   ```json
   {
     "rules": {
       "color-no-hex": true,
       "color-named": "never"
     }
   }
   ```
3. **动态样式处理**：如组件有 `:style="{ color: xxx }"` 动态样式，应使用变量：
   ```vue
   <!-- ✅ 正确 -->
   <div :style="{ color: 'var(--theme-primary)' }">

   <!-- ❌ 避免 -->
   <div :style="{ color: '#c20c0c' }">
   ```

### 9.5 风险点检查清单
